Something Confused in SBT and the build.sbt File

Nov 13th, 2013 | Comments

I created a sbt project, in build.sbt, I set the scalaVersion to 2.10.2, as I want to learn akka, I add the akka dependency:

libraryDependencies += "com.typesafe.akka" % "akka-actor" % "2.2.3"

It worked fine!

Then I want to write some code to try Spark, so I add

libraryDependencies += "org.apache.spark" % "spark-core_2.9.3" % "0.8.0-incubating"

When I use the ‘sbt update’ command to update the dependencies, it broken. Now I know as it indicated in the artifactId, it’s for scala 2.9.3, as it’s the latest version of spark.

I should change the scalaVersion AND I NEED TO change akka-actor’s version, as the akka-actor.2.2.3 is for scala 2.10, the 2.0.5 is for scala 2.9.

So finally, I need to change the scalaVersion to 2.9.3, change the akka-actor version to 2.0.5, and it works fine now.

sbt-doc.1

Nov 11th, 2013 | Comments

Install sbt and import sbt project to eclipse

download sbt-launch.jar, create a script file sbt with content:

SBT_OPTS="-Xms512M -Xmx1536M -Xss1M -XX:+CMSClassUnloadingEnabled -XX:MaxPermSize=256M"
java $SBT_OPTS -jar `dirname $0`/sbt-launch.jar "$@"

make the script executable

chmod u+x sbt

put the directory which contains sbt in PATH env var.

change sbt project to maven project

sbt make-pom

there will be a file in target directory, file name end with pom, copy it to project directory and rename it pom.xml. eclipsify the project:

mvn eclipse:eclipse

import the project to eclipse
