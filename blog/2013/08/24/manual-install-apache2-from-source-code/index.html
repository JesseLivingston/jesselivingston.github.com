Manual Install Apache2 From Source Code

Aug 24th, 2013 | Comments

1. Download apache source code

2. Install

2.1

If you encounter problem like “apr not found”, download the apr source code, extract it, ./configure, make, sudo make install

2.2

If you encounter problem like “apr-utils not found”, download the apr-utils source code, extract it, make, sudo make install.

2.3

If you encounter problem like pcre not found, download the pcre, extract it, make, sudo make install.

then cd into apache source code directory, configure apache again.

then

3. Startup

for convenience, add apache2/bin to PATH.

startup the server

3.1

If you encounter prolblem like libpcre.so.1 not found

a.

check the httpd dependencies:

it will show you the dependencies, find the libpcre.so.1 on you disk, suppose it’s under the /usr/local/lib directory, add a libc.conf file under /etc/ld.so.conf.d directory, add/append the path /usr/local/lib to the end of the file.

If you found there is already a libc.conf file under /etc/ld.so.conf.d directory, skip this step to next.

b.

Then startup the server.

If you encounter the problem like apachectl command not found, replace the apachectl with it’s absolutely path.
